Common FlutterFlow Bug Atlas

🔀 Broken Navigation Routes
HIGH
Pages push to incorrect routes due to wrong parameter passing or conditional logic errors in action chains. App may loop back to a previous screen or navigate to a blank page.
Fix Approach
Audit all page navigation actions. Verify parameter types match. Add null-safety guards on route conditions.
🔌 API Response Mapping Errors
HIGH
REST API response fields are mapped to wrong variable types. Causes null data on screens, app crashes, or silently empty UI components after a successful API call.
Fix Approach
Inspect each API call action. Validate JSON path mapping. Test with Postman then re-map in FlutterFlow.
🔐 Firestore Rule Conflicts
HIGH
Security rules block legitimate reads/writes causing permission-denied errors in production that didn't appear during development due to open test rules.
Fix Approach
Review Firestore rules in Firebase console. Test rules simulator per user role. Apply least-privilege principle.
🔄 Widget Rebuild Loops
MED
State variables trigger infinite widget rebuilds. App appears to freeze or shimmer. Common when a page's setState action is placed inside a build-time listener.
Fix Approach
Identify setState calls in backend queries. Move to conditional triggers. Use one-time loads where possible.
🔑 Auth Redirect Loops
HIGH
Authentication flow sends logged-in users back to the login screen or fails to persist session state across app restarts, forcing repeated login.
Fix Approach
Audit initial page auth condition. Verify Firebase Auth persistence. Check token refresh logic in actions.
🔔 Push Notification Failures
MED
FCM tokens not properly stored, iOS entitlements missing, or notification permission not requested, leading to notifications silently failing on device.
Fix Approach
Verify APNs certificate on iOS. Confirm FCM token save action fires post-login. Test via Firebase console.
🐌 Slow Page Rendering
LOW
Pages load slowly due to unoptimized Firestore queries, loading entire collections instead of paginated queries, or heavy widgets with synchronous data fetching.
Fix Approach
Add query limits & indexes. Use skeleton loaders. Convert collection queries to document-level queries.
⚙️ Build Configuration Errors
HIGH
Missing Google Services file, incorrect bundle IDs, or signing certificate mismatches cause builds to fail at the Xcode/Gradle level before the app even runs.
Fix Approach
Verify GoogleService-Info.plist and google-services.json placement. Validate bundle IDs match Firebase project.

Widget Tree & Action Audit

Systematically walk through every page in FlutterFlow. Inspect each widget's conditional visibility, data bindings, and action chains. Identify dead references, unused variables, and broken component links.

Backend Connection Review

Validate all Firebase integrations — Firestore collections schema, document references, sub-collections, and field types. Check for schema drift between what FlutterFlow expects and what exists in the database.

Firestore Schema Collections Document Refs Field Types
03

Firebase Security Rules Validation

Use the Firebase Rules Simulator to test read/write access for each user role. Fix overly permissive or overly restrictive rules. Ensure production rules match the application's actual access patterns.

Security Rules Role-Based Access Rules Simulator
04

Authentication Flow Testing

Test every auth entry point: email/password login, social OAuth, session persistence, token refresh, and logout. Validate that navigation correctly routes authenticated vs unauthenticated users.

Firebase Auth Session Persistence OAuth Token Refresh
05

API Response Inspection

Test every external API call via Postman or FlutterFlow's built-in API tester. Verify response structure matches the variable mapping. Fix null-safety issues and add error state handling for failed calls.

Mobile Testing Strategy

🍎 iOS — TestFlight

  1. Export Flutter project from FlutterFlow
  2. Open in Xcode, set bundle identifier
  3. Configure signing with Apple Developer cert
  4. Add GoogleService-Info.plist to target
  5. Enable Push Notifications entitlement
  6. Build archive (Product → Archive)
  7. Upload to App Store Connect via Xcode Organizer
  8. Add internal testers via TestFlight
  9. Test all flows on physical iPhone
  10. Verify FCM notifications on device

🤖 Android — Play Store Internal

  1. Add google-services.json to android/app/
  2. Configure applicationId in build.gradle
  3. Generate release keystore with keytool
  4. Configure signing in build.gradle
  5. Run flutter build appbundle --release
  6. Upload .aab to Play Console
  7. Set up internal testing track
  8. Add testers via Google email
  9. Test on physical Android device
  10. Verify FCM token & notifications
